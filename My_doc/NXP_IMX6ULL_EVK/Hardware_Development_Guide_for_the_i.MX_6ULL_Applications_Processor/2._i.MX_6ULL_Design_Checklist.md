# 2. i.MX 6ULL Design Checklist

This document provides a design checklist for the i.MX 6ULL processor.

The design checklist tables contain recommendations for the optimal design. Where appropriate, the checklist tables also provide an explanation of the recommendation so that you have a greater understanding of why certain techniques are recommended. All supplemental tables referenced by the checklist appear in the sections following the design checklist tables.

---

## 2.1. Design Checklist Tables

### Table 4. DDR Recommendations

| # | Recommendation | Explanation / Supplemental Recommendation |
|---|---|---|
| 1 | Connect the ZQPAD to an external 240 Ω 1% resistor to GND. | This is a reference used during the DRAM output buffer driver calibration. |
| 2 | Connect the DRAM_VREF to a source that is 50% of the NVCC_DRAM voltage value. | You may tie the DDR_VREF to a precision external resistor divider. Shunt the DDR_VREF to GND with a closely-mounted 0.1 μF capacitor. See Table 15 for resistor values. Using resistors with the recommended tolerances ensures the ±2% DDR_VREF tolerance per the DDR3 specification. You may use PMIC's tracking regulator, as used on the NXP reference designs. |
| 3 | Connect the DRAM_RESET to a 10 kΩ 5% pulldown resistor to GND. | **DDR3:** DRAM_RESET must be pulled down to meet the JEDEC sequence until the controller is configured and starts driving. The DRAM_RESET must be kept high when the DDR3 enters self-refresh mode. **LPDDR2:** DRAM_RESET must be left unconnected. Some NXP reference designs use a 1% resistor simply to consolidate the BOM. The DRAM_RESET is an active-low signal. |
| 4 | The DRAM_SDCKE0 and DRAM_SDCKE1 require external pull-down resistors to GND for JEDEC compliance when using LPDDR2. | **LPDDR2:** The SDCKE[1:0] must be pulled down to meet the JEDEC sequence until the controller is configured and starts driving. NXP designs use 10 kΩ. **DDR3:** The SDCKE[1:0] pull-down is not required to meet the JEDEC, unless deep-sleep or standby modes are used (see point 5). |
| 5 | The DRAM_SDCKE0 and DRAM_SDCKE1 require external resistors (such as 10 kΩ) to GND to minimize current drain during the Deep-Sleep Mode (DSM). | The BSP uses a common DDR routine for both fly-by and T-topology designs. Fly-by designs have a parallel resistor termination on address lines, while T-topology designs do not. During low-power self-refresh, the BSP programs the pad control register `GRP_CTLDS` to `0x00000000`, forcing DRAM_SDCKE0, DRAM_SDCKE1, and associated GRP_CTLDS I/O to the high-impedance state. External pull-down resistors are required to avoid floating outputs during standby. NXP designs use 10 kΩ. Any other termination on the DRAM_SDCKE0 and DRAM_SDCKE1 lines (such as 50 Ω) must not be present; simulation must be performed to ensure CKE signal integrity. |
| 6 | Make sure that a correct LPDDR2 function is connected to a correct I/O. Note that this does not necessarily correspond to the I/O name. | The MMDC IO names are for DDR3 by default. When LPDDR2 is selected, the I/O name (DDR3 MMDC PAD) does not match LPDDR2 functionality. See the "LPDDR2 and DDR3 pin mux mapping" table in the "Multi Mode DDR Controller (MMDC)" chapter of the chip reference manual. |

### Table 5. LCD Recommendations for Developer's Boot Modes

| # | Recommendation | Explanation / Supplemental Recommendation |
|---|---|---|
| 1 | When the LCD boot signals are used as the system's LCD signals, the other functions (or GPIO outputs after the boot) use a passive resistor network to select the desired boot mode for development boards. | Because only resistors are used, the LCD bus loads can cause current drain leading to higher (false) supply current measurements. Each LCD boot signal must connect to a series resistor to isolate the bus from the resistors and/or switchers (see Figure 1). Each configured LCD boot signal sees either a 10 kΩ pull-down or 10 kΩ pull-up. An alternate approach using buffers is implemented in the i.MX 6ULL EVK development board design. Either implementation is acceptable. |
| 2 | To reduce incorrect boot-up mode selections, perform one of these options: (a) Use the LCD boot interface lines only as processor outputs — ensure lines are not loaded down so the level is not misread during power-up. (b) If the LCD boot signal must be configured as an input, isolate the LCD signal from the target driving source with an analog switch and apply the logic value with a second analog switch. Alternately, peripheral devices with three-state outputs may be used; ensure the output is high-impedance during the boot-up interval. | Using LCD boot interface lines as inputs may result in a wrong boot-up due to the source overcoming the pull resistor value. If the usage of the LCD boot signal affects the peripheral device, then either an analog switch, an open collector buffer, or an equivalent must isolate the path. A pull-up or pull-down resistor at the peripheral device may be required to maintain the desired logic level. |
| 3 | The BOOT_CFG signals are required for proper functionality and operation and must not be left floating. | See the "System Boot" chapter in the chip reference manual for correct boot configuration. Note that an incorrect setting may result from an improper booting sequence. |

### Table 6. Boot Mode Input Recommendations

| Recommendation | Explanation / Supplemental Recommendation |
|---|---|
| For BOOT_MODE1 and BOOT_MODE0, use one of these options to achieve **logic 0**: tie to GND through an external resistor of any value, or tie directly to GND. For **logic 1**: tie directly to the VDD_SNVS_IN rail, or tie to VDD_SNVS_IN through a 10 kΩ external resistor (4.7 kΩ preferred in high-noise environments). ### Table 7. I²C Recommendations

| # | Recommendation | Explanation / Supplemental Recommendation |
|---|---|---|
| 1 | Verify the target I²C interface clock rates. | The bus can only operate as fast as the slowest peripheral on the bus. If faster operation is required, move slow devices to another I²C port. |
| 2 | Verify that the target I²C address range is supported and does not conflict with other peripherals. If there is an unavoidable address conflict, move the offending device to another I²C port. | These chips support up to four I²C ports. If it is undesirable to move the conflicting device, review the peripheral operation to see if it supports remapping of the address. |
| 3 | Do not place more than one set of pull-up resistors on the I²C lines. | This can result in excessive loading. A good design practice is to place one pair of pull-ups only. |

### Table 8. JTAG Recommendations

| # | Recommendation | Explanation / Supplemental Recommendation |
|---|---|---|
| 1 | Do not add external pull-up or pull-down resistors on the JTAG_TDO. | JTAG_TDO is configured with an on-chip keeper circuit so that the floating condition is actively eliminated. An external pull resistor on JTAG_TDO is detrimental. See Table 16 for a summary of the JTAG interface. |
| 2 | Ensure that the on-chip pull-up/pull-down configuration is followed when external resistors are used with JTAG signals (with the exception of JTAG_TDO). For example, do not use an external pull-down on an input that has an on-chip pull-up. | External resistors can be used with all JTAG signals except JTAG_TDO, but they are not required. See Table 16 for a summary of the JTAG interface. |
| 3 | JTAG_MOD (also referred to as SJC_MOD in some documents) must be externally connected to GND for normal operation. A pull-down resistor to GND is allowed; use 4.7 kΩ. | When JTAG_MOD is low, the JTAG interface is configured for common software debug, adding all system TAPs to the chain. When high, the JTAG interface is configured to a mode compliant with IEEE 1149.1. |

### Table 10. Power and Decoupling Recommendations

| # | Recommendation | Explanation / Supplemental Recommendation |
|---|---|---|
| 1 | Comply with the power-up sequence guidelines (as described in the data sheet) to guarantee reliable operation of the device. | Any deviation from these sequences may result in: excessive current during the power-up phase, prevention of the device from booting, or irreversible damage to the processor (worst-case scenario). |
| 2 | Do not overload the coin-cell backup power rail VDD_SNVS_IN. Associated I/Os (most have on-chip pull resistors and do not require external resistors): `POR_B` (configurable on-chip pull-up), `ONOFF` (on-chip pull-up), `BOOT_MODE0` (on-chip pull-down), `BOOT_MODE1` (on-chip pull-down), `TAMPER` (on-chip keeper), `PMIC_STBY_REQ` (configurable output), `PMIC_ON_REQ` (push-pull output), `TEST_MODE` (on-chip pull-down). | When `VDD_SNVS_IN = VDD_HIGH_IN`, the SNVS domain current is drawn from both equally. When `VDD_HIGH_IN > VDD_SNVS_IN`, VDD_HIGH_IN supplies all SNVS domain current and charges the coin-cell battery. When `VDD_SNVS_IN > VDD_HIGH_IN`, VDD_SNVS_IN supplies current to the SNVS and some flows into VDD_HIGH_IN. Note: VDD_HIGH_IN must be above the internal detector threshold (typically 2.4 V) for current flow to occur. If VDD_SNVS_IN is not powered by a battery, it is recommended that `VDD_SNVS_IN = VDD_HIGH_IN`. |
| 3 | Maximum ripple voltage limitation. | The common limitation for ripple noise must be less than 5% Vp-p of the supply voltage average value. Affected rails: all VDD_xxx_IN and VDD_xxx_CAP. |
| 4 | If VDD_SNVS_IN is directly supplied by a coin cell, a Schottky diode is required between VDD_HIGH_IN and VDD_SNVS_IN (cathode connected to VDD_SNVS_IN). Alternatively, VDD_HIGH_IN and VDD_SNVS_IN can be tied together if the real-time clock function is not needed during system power-down. | When no power is supplied to VDD_SNVS_IN, the diode limits the voltage difference between the two on-chip SNVS power domains to approximately 0.3 V. |

### Table 11. Oscillator and Clock Recommendations

| # | Recommendation | Explanation / Supplemental Recommendation |
|---|---|---|
| 1 | The precision 32.768 kHz oscillator connects a crystal between RTC_XTALI and RTC_XTALO. Choose a crystal with a maximum of 100 kΩ ESR and follow the manufacturer's recommendation for loading capacitance. Do not use an external biasing resistor — the bias circuit is on-chip. | The capacitors on either side of the crystal are about twice the crystal load capacitance. Typically 15–16 pF is employed to compensate for board and chip parasitic capacitance. Care must be taken to limit parasitic leakage from RTC_XTALI and RTC_XTALO to power or ground (>100 MΩ). Use short traces between the crystal and the processor with a ground plane under the crystal. |
| 2 | External kHz source: if feeding an external clock, RTC_XTALI can be driven DC-coupled with RTC_XTALO floated or driven with a complementary signal. | The voltage level must not exceed VDD_SNVS_CAP and the frequency must be <100 kHz under typical conditions. RTC_XTALI must not be driven if the VDD_SNVS_CAP supply is off. Note: if this external clock stops, the internal ring oscillator starts automatically. |
| 3 | An on-chip loose-tolerance ring oscillator of approximately 40 kHz is available. If RTC_XTALI is tied to GND and RTC_XTALO is floating, the on-chip oscillator is automatically engaged. | When high-accuracy real-time clock is not required, the system may use the on-chip 40 kHz oscillator (tolerance ±50%). The ring oscillator starts faster than the external crystal and is also used until the external crystal reaches stable oscillation. |
| 4 | The precision 24 MHz oscillator connects a fundamental-mode crystal between XTALI and XTALO. An 80 Ω typical ESR crystal rated for a maximum drive level of 250 μW is acceptable. Alternately, a 50 Ω typical ESR crystal rated for 200 μW may be used. See *i.MX 6 Series Crystal Drive (24 MHz)* (document EB830). | The NXP BSP software requires 24 MHz on this clock. This clock is used as a reference for USB and PCIe, so there are strict frequency tolerance and jitter requirements. See Table 17 for guidelines. |
| 5 | External MHz source: a single-ended external clock source can drive XTALI, with XTALO left externally floating. A differential external clock source can drive both XTALI and XTALO. | For XTALI VIL and VIH voltage levels, see the latest i.MX 6ULL datasheet. |
| 6 | CCM_CLK1_P / CCM_CLK1_N are LVDS input/output differential pairs compatible with TIA/EIA-644 standard, frequency range 0–600 MHz. Alternatively, a single-ended signal can drive CCM_CLKx_P with CCM_CLKx_N tied to a constant voltage equal to 50% of VDD_HIGH_CAP. If not used, any or both CCM_CLKx_N/P pairs may be left floating. | These clock inputs/outputs are general-purpose differential high-speed I/Os. They can be configured as inputs to feed external reference clocks to on-chip PLLs and/or modules, or as outputs used as a reference or functional clock for peripherals. |
| 7 | Bias the XTALI with a 2.2 MΩ resistor to GND. Mount the resistor close to the XTALI ball. | The XTALI bias must be adjusted externally to ensure a reasonable start-up time. |

## 2.6. Migrating from i.MX 6UltraLite to i.MX 6ULL

The main feature changes from the i.MX 6UltraLite to the i.MX 6ULL are adding the EPDC and ESAI support. The i.MX 6ULL's EPDC supports the E-book DC4 board.
